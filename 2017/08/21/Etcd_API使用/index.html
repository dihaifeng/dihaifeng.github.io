Etcd APi 使用指南

发表于 2017-08-21

Etcd的主要API是一个分层的关键空间。密钥空间由通常称为”节点”的目录和密钥组成

设置键的值

设置数据存储区中的第一个键值对。键是/message和值是Hello，World

响应对象包含几个属性：

  • action：刚刚提出的请求的动作。改请求尝试node.value通过PUTHTTP请求进行修改，因此操作的值为set
  • key：请求的HTTp路径。我们设定了/message的值为Hello world。所以key为/message。etcd使用类似文件系统的结构来表示键值对。因此所有的键都以/开头。
  • createdIndex：一个索引是为每个更改创建的唯一的，单调递增的整数。该特定索引反映了在创建给定键的etcd状态成员中的一点。您可能会注意到，在此示例中，索引2即使是您发送到服务器的第一个请求。这是因为内部命令也改变了幕后的状态，比如添加和同步服务器。
  • modifiedIndex：like node.createdIndex，这个属性也是一个etcd索引。这使数值变化的行动包括set，delete，update，create，compareAndSwap和compareAndDelete。由于get和watch命令不改变存储中的状态，它们不会更改modifiedIndex。

原子比较和交换

etcd可以用集群中的集中式协调服务，CompareAndSwap(CAS)用于构建分布式锁服务最基本的操作。
只有当客户端提供的条件等于当前条件时，此命令才会设置键的值

注意：CompareAndSwap对目录不起作用，如果尝试进行目录，将返回102，”不是文件的错误”

目前的比较条件是：

  • prevValue：检查键的上一个值
  • prevIndex：检查以前的modifyIndex的key
  • prevExist：检查key是否存在。如果prevExist是真，它是一个update请求；如果prevExist是false，它是一个create请求。

统计

一个etcd集群跟踪一些统计数据，包括延迟，带宽和正常运行时间。这些通过统计端点公开，以了解集群的内部运行状况。

  • leader统计

leader可以看在到整个集群，并跟踪两个有趣的统计信息：集群中每个对等体的延迟以及失败和成功的Raft RPC请求数。可以从/v2/stats/leader端获取这些统计信息：

  • Self统计

每个节点保留一些内部统计信息：

  • Store统计
    Store统计信息包括有关节点处理的操作的信息。请注意，v2store Statistics存储在内存中。当成员停止时，重新启动时将重新存储统计信息。
    整个集群都会看到修改Store的状态（如创建、删除、设置和更新）的操作，所有节点上的数量都会增加。像get和watch这样的本地操作时本地节点，只能在这个节点上看到。
